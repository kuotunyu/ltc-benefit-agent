# 恢復專案脈絡

## 流程

1. 讀取 `CLAUDE.md`、`AGENTS.md`、`PLAN.md` 與 `PROGRESS.md`；以 PROGRESS 快速回憶區掌握上次收工狀態。
2. 列出 workspace 檔案與目前 Phase 相關產物，不讀取 `.env` 內容、模型權重、logs 原始對話或其他秘密。
3. 對照 PLAN 的當前 Phase DoD，逐項標示「已有證據／尚未驗證／不適用」。不得把文件宣稱當成實跑證據。
4. 只有在專案骨架已存在且不會觸發付費或長時間模型工作時，才執行當前 Phase 定義的唯讀測試或 smoke check；否則清楚寫出未執行原因。
5. 只有下一步需要本機模型時才查 `nvidia-smi`、`ollama list` 與磁碟；不得停止程序、拉模型、占用連接埠或修改環境。
6. 回報目前 Phase、已完成證據、已知坑、待作者處理與一個具體下一步，等待作者指示。

## 不可違反

- 不執行任何 Git 指令，包括唯讀的 status、log 或 diff；Git 由作者本人操作。
- 不修改檔案、不修測試、不安裝套件；本 skill 只恢復脈絡。
- 不顯示絕對私密路徑、API key、`.env` 真值或含個資的原始 log。
- 發現 PROGRESS 與實際不符時，如實列出差異，不自行補寫成已完成。

## 回報格式

- 現況：Phase 與一句話狀態。
- 證據：最多五項實際檔案或命令結果。
- 風險／待作者處理：沒有就寫「無」。
- 建議下一步：一項可直接開始且不跨 Phase gate 的工作。
